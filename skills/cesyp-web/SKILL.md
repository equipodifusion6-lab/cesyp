# CESyP Institutional Editorial Skill

## Open Design Generation Skill

You are designing a modern institutional platform for the Centro de Estudios Sociales y Políticos (CESyP) de Jujuy.

This is NOT:

* a startup website,
* a political campaign,
* a SaaS dashboard,
* a newspaper homepage,
* a government bureaucracy portal.

This IS:

* a territorial think tank,
* an editorial institutional platform,
* a public analysis organization,
* a modern civic observatory,
* a strategic research institution.

The design language must communicate:

* intellectual seriousness,
* institutional legitimacy,
* territorial presence,
* contemporary editorial quality,
* organizational maturity,
* calm authority.

---

# PRIMARY DESIGN GOAL

The platform should make users think:

"This organization is serious, organized, modern and intellectually credible."

Everything must reinforce:

* clarity,
* structure,
* readability,
* hierarchy,
* calm confidence.

The interface should feel:

* premium,
* editorial,
* strategic,
* minimal,
* elegant.

---

# VISUAL STYLE

## Core Style

Modern editorial institutional design.

Inspired by:

* international think tanks,
* contemporary editorial publications,
* cultural institutions,
* modern research organizations.

The aesthetic should combine:

* typography-driven layouts,
* whitespace,
* strong hierarchy,
* restrained interactions,
* subtle motion.

---

# TYPOGRAPHY RULES

Headlines:

* elegant,
* high contrast,
* large breathing space.

Body:

* highly readable,
* long-form optimized,
* relaxed line height.

Never:

* compress text,
* overcrowd layouts,
* use tiny metadata.

---

# LAYOUT SYSTEM

## Grid

Desktop:
12-column layout.

Reading width:
720–820px max.

Mobile-first implementation.

Use whitespace aggressively.

Do not create dense interfaces.

---

# EXPERIENCE PRINCIPLES

## Reading First

Content is the hero.

The interface exists to:

* improve reading,
* structure information,
* increase trust,
* encourage exploration.

---

## Calm Complexity

The platform may contain large amounts of information,
but the interface must always feel:

* ordered,
* breathable,
* understandable.

---

# COMPONENT STYLE

## Buttons

Primary:

* black background,
* white text,
* subtle radius,
* medium weight.

Secondary:

* neutral surface,
* soft borders,
* understated interaction.

Avoid:

* glowing buttons,
* gradients,
* excessive animation.

---

## Cards

Cards should feel:

* editorial,
* spacious,
* structured.

Use:

* subtle borders,
* minimal shadows,
* strong typography.

Avoid:

* dashboard aesthetics,
* floating random cards,
* excessive depth.

---

# MOTION SYSTEM

Motion must be:

* subtle,
* short,
* purposeful.

Allowed:

* opacity transitions,
* slight translate,
* smooth hover states.

Avoid:

* bounce,
* elastic effects,
* exaggerated transforms.

---

* immersive,
* editorial,
* serious.
